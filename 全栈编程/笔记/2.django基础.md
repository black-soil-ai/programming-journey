### 3.URL调度器

#### 3.1URL调度器简单介绍

- 这个模块是纯粹的 Python 代码，包含 URL 模式（简单的正则表达式）到 Python 函数（视图）的简单映射。
- URL 调度器也被称为 URL 路由或者 URLConf 或者 URL 路由分发器。
```bash
# 配置路由（path()函数）
path('hello/',views.hello),
# 设计视图（视图函数）
from django.http import HttpResponse
def hello(request):
	return HttpResponse('Hello,Django')
	
# 路径转化器（除了int还有str,slug等等）
path('page/<int:pg>/', views.page_view),
# 对应的视图
def page_view(request, pg):  # 这里的 pg 必须和路由里的 <int:pg> 名字一致
    return HttpResponse(f"你访问的是第 {pg} 页")
```

#### 3.2 re_path()函数

- re_path()函数比path()函数更加灵活

- ```python
  from django.urls import path, re_path, include
  re_path(r'page/(\d{4})', views.repath  # 默认的数据类型是str
  ```

  

#### 3.3路径转换相关知识点和总结

> 总结 使用URL调度器
>
> 第一步： urls.py 中定义路由，并且指定视图函数 
>
> 第二步： views.py 中定义视图函数并返回 HttpResponse 响应 
>
> 第三步：在浏览器直接输入地址或者在视图函数中响应 HTML 链接( a 标签)到客户端，然后进行访问

### 4.应用

#### 4.1简介

将一个复杂的项目拆分成一个个小的清晰的模块（应用app），有利于维护和使用

#### 4.2创建应用

```python
# 创建应用
python manage.py startapp name
# 注册应用（在setting.py中设置）
INSTALLED_APPS{
	...
	'app01'
	...
}
应用的目录在上方展示（序号0）
```

#### 4.3include( )函数

``` python
# 进行路由分发
from django.urls import include
# 将/user/请求，发送给user应用的urls.py处理 
path('user/', include('user.urls'))
# 由此形成分布式url调度器
```

### 6.便捷视图函数(调用模板)

#### 6.1render()函数

- 在urls文件里进行设置path（）映射到视图，视图里的函数在返回模板里的html文件

- ```python
  from django.shortcuts import render
  
  def test(request):
      return render(request, 'test.html')
  ```

  

#### 6.2redirct()函数 又叫做重定向

- 跳转到指定的网址路由

- ```python
  from django.shortcuts import redirect
  def test(request):
      return redirect('https://www.baidu.com')
      
  ```
