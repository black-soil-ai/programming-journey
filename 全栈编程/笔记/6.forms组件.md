## 注册功能



当我们在前端中写form表单时，是如何完成检测用户填入的内容是否符合规范呢？





```python
# 类似于模板 需要创建文件forms.py组件
# 导入forms组件
from django import forms
# 继承forms
class my_forms(forms.Form):
    username=forms.CharField(min_length=3,max_length=8)

    password=forms.CharField(min_length=3, max_length=8)

    email=forms.EmailField()
    
    
    
# 然后进入text.py进入测试 
from app01 import forms

    # 实例化对象（以字典的形式进行传递）
    myforms = forms.my_forms({'username': 'aaa', 'password': '111', 'email': 'zhangxiyu@qq.com'})

    # 所有数据是否正常初始化
    print(myforms.is_valid())
    # 查看正常初始化的数据
    print(myforms.changed_data)
    # 没通过的数据
    print(myforms.errors)

    
```

## 功能展示

```python 
# 进行校验（后端）
def form(request):

    # 先创建一个forms对象
    myforms=forms.my_forms()

    if request.method == 'POST':
        # 对传来的数据进行校验
        myforms=forms.my_forms(request.POST)
        print(myforms.changed_data)
        if myforms.is_valid():
            return HttpResponse('ok')



    return render(request,'form.html',locals())




# 对前端进行渲染（有3种方法）
{#    关闭表单自带的验证功能   #}
<form  method="post" action=""  novalidate >
{% for myform in myforms %}

    <p>{{ myform.label }}:{{ myform }}<span style="color : red">{{ myform.errors.0 }}</span></p>

{% endfor %}
<input type="submit">
</form>
```

## forms组件的参数



1. **通用参数（适用于大多数字段）**

- label (str): 字段的标签（在表单中显示的文本）。如果未提供，Django 会根据字段名自动生成 （将下划线替换为空格并首字母大写）。
- required (bool): 字段是否必须填写（默认为 True）。如果设置为 False，则字段可以为空。
- initial (any): 字段的初始值（当表单未绑定数据时显示）。
- help_text (str): 字段的帮助文本（显示在字段下方，用于说明）。
- error_messages (dict): 自定义错误消息。可以覆盖默认的错误消息。字典的键是错误类型（如 'required', 'invalid', 'max_length' 等），值是要显示的错误消息字符串。
- validators (list): 自定义验证器函数列表。每个验证器是一个可调用对象，接收字段值并验证， 如果验证失败则抛出 ValidationError 。
- disabled (bool): 如果为 True，则字段在表单中显示为禁用状态（用户无法编辑）。
- widget (Widget): 指定渲染字段的 Widget 类。通过 Widget 可以控制字段在 HTML 中的渲染方式 （如输入框、下拉框等）以及 HTML 属性（如 class, placeholder 等）。
- localize (bool): 是否本地化字段（用于日期、数字等字段的本地化显示和输入）。
- label_suffix (str): 覆盖表单的 label_suffix （标签后缀，默认为冒号 ":"）。

2. **特定字段的参数**

字符类型字段（CharField, EmailField, URLField, 等）

- max_length (int): 字段的最大长度（字符数）。如果设置了，Django 会验证输入的长度不超过此 值。
- min_length (int): 字段的最小长度（字符数）。如果设置了，Django 会验证输入的长度不小于此 值。
- strip (bool): 是否去除输入的前后空白（默认为 True）。对于 CharField ，通常建议保留为 True。

数值类型字段（IntegerField, DecimalField, FloatField）

- max_value (int/float): 允许的最大值。
- min_value (int/float): 允许的最小值。

日期时间字段（DateField, DateTimeField, TimeField）

- input_formats (list): 可接受的日期/时间格式列表（字符串格式）。例如： ['%Y-%m-%d', '%m/%d/%Y'] 。

选择字段（ChoiceField, MultipleChoiceField）

- choices (iterable): 字段的选项。通常是一个由二元组（值, 显示文本）组成的列表或元组。例 如： [('M', 'Male'), ('F', 'Female')] 。

文件字段（FileField, ImageField）

- max_upload_size (int): 上传文件的最大大小（字节）。注意：实际的文件大小验证通常需要在 视图中进行，因为表单验证可能无法完全控制
- allow_empty_file (bool): 是否允许空文件（默认为 False）。
